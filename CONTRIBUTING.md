# Contributing

Thanks for taking a look. This repo is the public docs + OpenAPI spec for the API at `api.pytheum.com`. The API server itself is closed-source for now; everything here is fair game.

## Found a bug?

File an issue. The templates make this easy — pick "Bug report" and fill in the four sections. A `curl` one-liner that reproduces the problem is worth more than any amount of prose.

## Want a new endpoint or behavior?

Open a feature-request issue first. We want to talk before you write code. The roadmap is reactive to actual usage, so describing the use case is the most useful thing you can do.

## Pull requests

PRs welcome for:

- Docs typos and clarifications
- `openapi.yaml` fixes (missing fields, wrong types, broken examples)
- New examples under `examples/` or `docs/quickstart/`

Style: terse, dev-respecting, no marketing voice. Match the existing [README.md](README.md) and [ROADMAP.md](ROADMAP.md). Keep lines short, keep claims testable.

For anything larger than a typo, please link the issue your PR closes.

## Code of conduct

Participation is governed by [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). Report conduct issues to `conduct@pytheum.com`.

## Security

Don't file public issues for security problems. See [SECURITY.md](SECURITY.md).
